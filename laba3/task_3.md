# Задание 3. Система синхронизации и отчётов по курсу чешской кроны

## Контекст

Чешский национальный банк (ČNB) предоставляет данные о курсе чешской кроны в двух видах:

- **Ежедневный курс**:  
  https://www.cnb.cz/en/financial_markets/foreign_exchange_market/exchange_rate_fixing/daily.txt?date=27.07.2019

- **Исторические данные за год**:  
  https://www.cnb.cz/en/financial_markets/foreign_exchange_market/exchange_rate_fixing/year.txt?year=2019

Для финансовых и аналитических приложений важно **автоматически получать актуальные данные**, хранить их в базе и строить **отчёты по выбранным валютам**.

---

## Задание

Разработать программное решение, которое обеспечивает:

---

### 1. Автоматическая синхронизация данных

- Синхронизация данных по чешской кроне в БД должна выполняться **по расписанию**.
- Время или интервал запуска должны быть **конфигурируемыми** (например, каждый день в 0:01).
- Список валют для синхронизации должен быть **настраиваемым** в конфигурации.

---

### 2. Синхронизация за период времени

- Приложение должно позволять синхронизировать данные за заданный интервал:
    - входные параметры: `startDate` и `endDate`;
    - валюты для синхронизации — конфигурируемые.
- Система должна корректно обрабатывать пропуски и отсутствующие данные.

---

### 3. Web-API для отчётов

- Предоставить API для получения отчёта по курсу кроны за указанный период.
- В отчёте выводить для каждой выбранной валюты:
    - минимальное значение;
    - максимальное значение;
    - среднее значение.
- Показатели рассчитываются **для 1 условной единицы валюты** (`Amount = 1`).
- Формат отчёта — **JSON**.
- Валюты для отчёта передаются в запросе.
- Необходимо корректно обрабатывать аномалии данных (например, отсутствующие значения в некоторых датах).

---

## Требования к реализации

- Данные должны храниться в **персистентном хранилище** (СУБД, файл и т.п.).
- Приложение должно быть конфигурируемым:
    - время/интервал синхронизации;
    - список валют;
    - период синхронизации (для исторических данных).
- Обработка ошибок и пропусков в данных должна быть предусмотрена.
